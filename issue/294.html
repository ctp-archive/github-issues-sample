There is NO issue with the data from the website. Being my first time using ArduinoJSON with a much larger HTTP GET data set than previous experience, I just needed to learn and apply new techniques to solve the issue.

My issue was that the states JSON data just got too big for my MCU memory capacity. The unit would just spontaneously reboot when trying to deserialize the states data with a JSON capacity to fit the data.

I went back to the ArduinoJSON documentation to get smarter and discovered the filtering capability. This feature reduces memory usage significantly, in my case, thus avoiding crashes.

The filtering is a Very cool capability. Thanks ArduinoJSON authors.
